# h-2-1 영상 화질 개선 가이드

## 1. 웹에서 적용한 것 (코드)

- **preload="auto"**  
  페이지 로드 시 영상을 미리 불러와서 재생 시 더 선명하게 재생되도록 했습니다.
- **object-fit: contain**  
  영상을 비율에 맞게만 보여 주어, 늘어나면서 뿌옇게 보이는 것을 막았습니다.
- **transform: translateZ(0)**  
  GPU 가속을 쓰도록 해서 브라우저가 더 선명하게 그리도록 유도했습니다.
- **업스케일 방지**  
  작은 영상을 크게 키우지 않도록 해서 흐릿해지는 것을 줄였습니다.

## 2. 가장 중요한 것: 원본 영상 화질

웹에서 할 수 있는 건 “이미 있는 화질을 최대한 잘 보이게” 하는 정도입니다.  
**실제로 선명하게 보이려면 원본 파일 자체가 고화질이어야 합니다.**

### 권장 인코딩 (h-2-v1.mp4, h-2-v2.mp4 교체 시)

| 항목 | 권장 |
|------|------|
| 해상도 | **1920×1080 (Full HD)** 이상 |
| 코덱 | **H.264 (AVC)** |
| 비트레이트 | **8~12 Mbps** (고화질), 최소 5 Mbps |
| 프레임 | 30fps 또는 60fps |

### 예시 (FFmpeg로 고화질 인코딩)

```bash
ffmpeg -i 원본.mp4 -c:v libx264 -preset slow -crf 18 -b:v 10M -c:a aac -b:a 192k h-2-v1.mp4
```

- `-crf 18`: 화질 우선 (숫자 낮을수록 고화질, 18~23 권장)
- `-b:v 10M`: 비디오 비트레이트 10 Mbps

### root index 영상이 더 선명하게 보이는 이유

- root의 `h-1.mp4`, `h-2.mp4`가 **해상도·비트레이트가 더 높게** 인코딩되어 있을 가능성이 큽니다.
- h-2-v1, h-2-v2를 **같은 해상도·비트레이트로 다시 인코딩**해서 교체하면, carplatform-ver2에서도 비슷하게 선명하게 보입니다.

## 3. 정리

1. **지금 단계**:  
   - `preload="auto"`, `object-fit: contain`, GPU 레이어 등은 이미 적용되어 있음.
2. **추가로 할 일**:  
   - **h-2-v1.mp4, h-2-v2.mp4를 Full HD + 8~10 Mbps 정도로 다시 인코딩**해서 `images/h-2/` 폴더에 덮어쓰기.
3. **root와 동일한 느낌으로 하려면**:  
   - root의 `h-1.mp4`, `h-2.mp4`와 **같은 해상도·비트레이트**로 h-2-v1, h-2-v2를 맞춰 주면 됩니다.
